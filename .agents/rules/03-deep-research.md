# Rule 3: Mandatory Deep Web Research (DuckDuckGo)

## 1. Zero Hallucination Doctrine
You MUST NEVER hallucinate package versions, architectural patterns, deprecated functions, or security vulnerabilities. If you do not have 100% certainty, you MUST perform a deep web search.

## 2. DuckDuckGo Usage (`mcp_duckduckgo-search_search`)
For ANY unknown variable, library update, or system implementation (like eBPF or Bubbletea nuances), you must use the `duckduckgo-search` tool.
- Query explicitly for the latest 2026 data.
- Do not guess the Go SDK signatures.
- If a web search fails or returns useless data, adjust your query and search again.

## 3. Evidence First Pattern
Your decisions MUST be backed by search results. No search = No code execution.
